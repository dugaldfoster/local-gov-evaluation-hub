    <section class="main">
      <h2 class="section-title">Start here</h2>
      <p class="section-lede">
        If you’re not sure where to begin, use this quick path: clarify decisions → map outcomes → choose a feasible design → plan data → do analysis → communicate and act.
      </p>

      <div class="grid">
        <article class="card" data-card>
          <h3>1) Define the decision</h3>
          <p>What will change depending on the findings? Who needs confidence, and how much?</p>
          <ul>
            <li>Write a one-sentence decision statement (“We will decide whether to…”)</li>
            <li>Agree a timeframe and what “good enough” looks like</li>
            <li>List constraints (budget, ethics, data access, politics)</li>
          </ul>
          <div class="meta">
            <span class="tag">Scope</span><span class="tag">Stakeholders</span><span class="tag">Governance</span>
          </div>
        </article>

        <article class="card" data-card>
          <h3>2) Map outcomes and assumptions</h3>
          <p>Turn a programme description into a testable story (logic model / theory of change).</p>
          <ul>
            <li>Inputs → activities → outputs → outcomes</li>
            <li>Note risks, inequalities, and who benefits</li>
            <li>Identify measurable signals you can feasibly collect</li>
          </ul>
          <div class="meta">
            <span class="tag">Theory of change</span><span class="tag">Logic model</span>
          </div>
        </article>

        <article class="card" data-card>
          <h3>3) Choose a design that fits reality</h3>
          <p>Pick the strongest approach you can deliver, then be honest about limitations.</p>
          <ul>
            <li>Process evaluation for implementation + learning</li>
            <li>Impact evaluation for attribution / contribution</li>
            <li>Value for money when costs and trade-offs matter</li>
          </ul>
          <div class="meta">
            <span class="tag">Methods</span><span class="tag">Feasibility</span>
          </div>
        </article>

        <article class="card" data-card>
          <h3>Common pitfalls (and fixes)</h3>
          <p>Fast checks that prevent wasted effort.</p>
          <ul>
            <li><strong>Pitfall:</strong> measuring what’s easy, not what matters → <strong>Fix:</strong> link every metric to a decision</li>
            <li><strong>Pitfall:</strong> “evaluation” starts too late → <strong>Fix:</strong> minimum viable baseline plan</li>
            <li><strong>Pitfall:</strong> findings go nowhere → <strong>Fix:</strong> action log + owner + date</li>
          </ul>
          <div class="meta">
            <span class="tag">Quality</span><span class="tag">Delivery</span>
          </div>
        </article>

        <article class="card" data-card>
          <h3>Templates you can copy into Word</h3>
          <p>Use these as headings in a short evaluation plan.</p>
          <ul>
            <li>Purpose and key questions</li>
            <li>Design and comparison</li>
            <li>Data sources and ethics</li>
            <li>Analysis plan (brief)</li>
            <li>Reporting and decision timeline</li>
          </ul>
